# Sprint 2 Strategy & Backlog

## Sprint Goal
Execute Phase 1 & 2 of the Practical Ranking Roadmap for two more top-priority topics: **Utility Cost** and **Water Planning**. Building these two pillar Hub Pages will establish strong topical clusters for our existing calculators in these categories, improving crawlability, semantic relevance, and AI search extraction.

- [x] **Task 1:** Create `/docs/SPRINT_2_PLAN.md` (This document)
- [x] **Task 2:** Create the Hub Page component (`UtilityCostHub.tsx`) featuring Semantic HTML, FAQ, JSON-LD schema, and internal links to existing calculators (Propane, Internet, Solar, Peak Energy, Cable).
- [x] **Task 3:** Create the Hub Page component (`WaterPlanningHub.tsx`) featuring Semantic HTML, FAQ, JSON-LD schema, and internal links to existing calculators (Water Hauling/Fill, Septic, Well).
- [x] **Task 4:** Wire up `/utility-cost` and `/water-planning` routes in `App.tsx`.
- [x] **Task 5:** Add internal links to these Hubs on the Home page, and add return links inside the individual calculators back to their respective hubs.

## Rationale
Following the success of the Generator Hub, creating the Utility Cost Hub and Water Planning Hub will structure the remaining high-value rural decision problems into tight clusters. This hub-and-spoke model tells Google that RuralUtilityCost is the definitive library for these topics, while providing direct, structured answers for AI chatbots.
